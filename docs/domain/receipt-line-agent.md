# レシート読み取りLINE Agent

## 概要
LINEで送られたレシート画像を読み取り、同棲生活における共有支出の月末精算に必要な情報をGoogleスプレッドシートへ記録する。

本機能は単なる家計簿Botではなく、食費・日用品費などの立替精算を低負担で続けるための記録エージェントとして扱う。

## 背景・動機
同棲生活では、食費や日用品などの共有支出が日常的に発生する。月末に精算するためには、購入日、店舗、金額、レシート画像などの記録が必要になる。

手作業での転記は、記録漏れ、後回し、レシート紛失が起きやすい。そこで、普段使っているLINEへレシート画像を送るだけで、精算に必要な最低限の情報をスプレッドシートへ蓄積できる仕組みを作る。

## 目的
月末精算に必要な共有支出記録を、できるだけ低負担で残すことを目的とする。

具体的には、以下を実現する。

- レシート画像をLINEに送るだけで記録できる
- 食費・日用品を中心とした共有支出を記録できる
- 月末にスプレッドシートを見れば、精算対象の支出を確認できる
- レシート画像の参照先を残し、必要に応じて後から確認できる

## スコープ
MVPでは、家族・同居者など複数人が同じLINE Botへレシート画像を送る運用を前提とする。レシート画像を送信したLINEユーザーを支払者として記録する。

### 対象
- 同棲生活の共有支出
- 主な記録対象は食費と日用品
- LINEで送信されたレシート画像
- Googleスプレッドシートへの記録
- レシート画像のS3保存
- LINEテキスト質問による今月の簡易集計回答

### 対象外
- 月次集計の自動精算・定期配信
- 月指定、全期間、店舗別、カテゴリ別などの詳細集計質問
- 精算済み管理
- 精算対象外フラグ
- 分担率の管理
- 複数スプレッドシート振り分け
- 修正用の対話フロー
- 会計アプリ連携
- 管理画面

個人支出の厳密な除外はMVPでは自動判定せず、必要に応じてスプレッドシート上で手動修正する。

## 要件

### システムフロー

#### 基本フロー
1. 利用者がLINE Botへレシート画像を送信する。
2. Webhook LambdaがLINE署名を検証し、画像メッセージであることを確認する。
3. Webhook LambdaがLINE Messaging APIから画像バイナリを取得する。
4. Webhook Lambdaが送信者のLINEユーザーIDとLINE表示名を取得する。
5. Webhook Lambdaが画像をS3へ保存する。
6. Webhook Lambdaが受付メッセージをLINEへ返信し、SQSへ処理依頼を投入する。
7. Worker LambdaがSQSメッセージを受け取り、Agent Core Runtimeを呼び出す。
8. AgentがGoogle Cloud Vision APIでレシート画像からOCRテキストを抽出する。
9. AgentがLLMで抽出結果を補正し、記録用データへ正規化する。
10. 店舗名と合計金額が取得できた場合、Agentが支払者情報とともにレシート日付に対応する月別シートへGoogle Sheets APIで1行追記する。
11. Worker LambdaがLINE Push Messageで登録結果を通知する。

#### テキスト質問フロー
1. 利用者がLINE Botへ今月の集計に関するテキストメッセージを送信する。
2. Webhook LambdaがLINE署名を検証し、送信者のLINEユーザーIDとLINE表示名を取得する。
3. Webhook LambdaがLINEグループからのメッセージであることを確認し、LINE Secretの `allowedExpenseQuerySourceIds` で送信元グループIDが許可されていることを確認する。
4. Webhook LambdaがAgent Core Runtimeを同期呼び出しする。
5. AgentがLLMで質問意図を `self_total`、`overall_total`、`by_user_total`、`unsupported` のいずれかに分類する。
6. AgentがGoogle Sheets APIでJST現在月の `yyyy-MM` シートだけを読み取り、コードで合計金額を計算する。
7. Webhook LambdaがLINE Reply Messageで集計結果を返信する。

#### 例外フロー
1. 画像・テキストメッセージ以外を受信した場合、レシート画像または今月の集計質問を送るよう案内し、記録処理は行わない。
2. LINE署名検証に失敗した場合、不正なリクエストとして拒否し、記録処理は行わない。
3. LINE画像取得に失敗した場合、画像取得失敗としてLINEへ通知し、CloudWatch Logsへ記録する。
4. S3保存に失敗した場合、画像保存失敗としてLINEへ通知し、CloudWatch Logsへ記録する。
5. 店舗名または合計金額を読み取れない場合、スプレッドシートには登録せず、撮り直しまたは再送をLINEへ依頼する。
6. Google Sheetsへの転記に失敗した場合、読み取りはできたが転記に失敗したことをLINEへ通知し、エラー内容をCloudWatch Logsへ記録する。MVPでは自動再実行の管理画面や手動再実行機能は用意しない。
7. 同じLINEメッセージを再処理した場合、二重登録しない。
8. テキスト質問が対象外の場合、Agentは対応できる質問範囲をLINEへ返信し、Sheetsの更新は行わない。
9. 集計対象の月別シートが存在しない場合、Agentは今月の登録がない旨をLINEへ返信し、シート作成は行わない。
10. テキスト質問の送信元トークが許可されていない場合、Webhook Lambdaは集計できない旨を返信し、Agent Core Runtimeを呼び出さない。

### 機能要件

#### LINE連携
- LINE Webhookで画像メッセージを受け取れること。
- LINE署名検証を行い、正規のLINEリクエストのみ処理すること。
- Webhookでは受付返信を行い、読み取り結果はPush Messageで通知すること。
- テキスト質問ではWebhookから同期的にAgent Core Runtimeを呼び出し、Reply Messageで回答すること。
- テキスト集計質問はLINEグループからのみ受け付け、LINE Secretの `allowedExpenseQuerySourceIds` に含まれるグループIDからのみ許可すること。
- 受付返信と読み取り結果通知は、画像を受け付けた送信元会話へ返すこと。個人チャットでは送信者、グループまたはルームでは送信元のグループまたはルームへ返すこと。
- Webhookで取得できるMessaging APIのLINEユーザーIDと、LINEプロフィールAPIから取得したLINE表示名を支払者情報として扱うこと。グループからのメッセージではグループIDとユーザーIDを使い、グループメンバープロフィールAPIから表示名を取得すること。
- LINE表示名を取得できない場合でも、LINEユーザーIDが取得できていれば受付処理を継続し、表示名は空欄で記録すること。

#### レシート画像管理
- LINEから取得した画像をS3へ保存すること。
- S3バケットの一覧は公開せず、`receipts/*` の画像オブジェクトのみ公開URLで参照できること。
- スプレッドシートにはレシート画像の公開URLを記録し、後からクリックして確認できること。
- レシート画像は90日後に自動削除すること。

#### AI Agent処理
- Agent Core Runtime上のPython Agentでレシート処理を行うこと。
- Agent実装は新規最小構成で作成し、`agentcore-sample` の既存サンプル構成に不要に依存しないこと。
- Google Cloud Vision APIのOCRテキストを事実のベースとして扱うこと。
- LLMは店舗名、日付、カテゴリ、合計金額の補正・正規化に使うこと。
- LLMはClaude Haikuを利用すること。BedrockのモデルIDまたはInference Profile IDは環境変数で変更できること。
- 日付が不明な場合は、今日の日付で補完しないこと。
- 店舗名または合計金額が不明な場合は登録しないこと。
- テキスト質問ではLLMを質問意図の分類にのみ使い、集計値の計算はコードで行うこと。
- テキスト質問の分類は `self_total`、`overall_total`、`by_user_total`、`unsupported` の4種類とすること。
- 「現在のユーザー」「自分」は、質問したLINEユーザーIDとして扱うこと。

#### スプレッドシート記録
- Google Sheets APIで対象スプレッドシートへ1行追記すること。
- レシート日付に対応する月別シートを利用すること。
- 月別シートが存在しない場合は、同じカラム構成で新規作成してから追記すること。
- 月別シート名は `yyyy-MM` 形式とする。例: `2026-05`
- レシート日付が不明な場合は、処理日が属する月のシートへ登録し、レシート日付列は空欄にすること。
- Google Sheets認証はService Accountを利用すること。
- Service Accountの認証情報はAWS Secrets Managerで管理すること。
- Google SheetsのスプレッドシートID、Service Account認証情報のSecret ARN、LINE Botの認証情報は環境変数またはSecrets Manager経由で受け取れるようにすること。
- MVPのシート列は以下とする。

| 列 | 項目 | 例 |
|---|---|---|
| A | 登録日時 | 2026-05-07T22:00:00+09:00 |
| B | レシート日付 | 2026-05-07 |
| C | LINE表示名 | 太郎 |
| D | LINEユーザーID | U1234567890abcdef |
| E | 店舗名 | ローソン |
| F | カテゴリ | 食費 |
| G | 合計金額 | 1280 |
| H | 画像URL | https://receipt-images.s3.ap-northeast-1.amazonaws.com/receipts/user/message.jpg |
| I | LINEメッセージID | 1234567890 |

#### スプレッドシート集計
- 集計質問ではJST現在月の `yyyy-MM` シートのみを読み取ること。
- 読み取り範囲は `A:I` とし、LINE表示名、LINEユーザーID、合計金額を使って集計すること。
- 本システムは1つのLINEグループでのみ利用する前提のため、許可されたグループからの集計質問では現在月シートの全行を対象とすること。
- 集計ツールは読み取り専用とし、月別シートが存在しない場合でも新規作成しないこと。
- 合計金額が空欄または数値化できない行は集計対象外とすること。
- ユーザー別集計では内部キーをLINEユーザーIDとし、表示名はシート上のLINE表示名を使うこと。表示名が空の場合はLINEユーザーIDの短縮表記を使うこと。

#### カテゴリ
- MVPのカテゴリは食費、日用品、その他の3種類に固定すること。
- 食費または日用品に判定できない場合は、その他として扱うこと。
- カテゴリは将来の月末精算や集計に使える形で保存すること。

### 非機能要件
- インフラはAWS CDKで管理すること。
- CDK環境はまずprodのみとすること。
- LINE Webhookの入口はPublic API Gatewayとすること。
- OCR、LLM、Sheets転記はSQS経由で非同期処理すること。
- 処理失敗時に再試行できるよう、SQS DLQを用意すること。
- LINEメッセージIDをシートへ保存し、WorkerやAgentの再試行が発生しても二重登録を防ぐこと。
- LINEトークン、Google認証情報などの秘匿情報はSecrets Managerで管理すること。
- LINE BotとGoogle Sheetsは実装時点で未作成でもよい。デプロイ後に必要な値を環境変数またはSecrets Managerへ設定できる構成にすること。
- 処理結果や失敗理由はCloudWatch Logsで追跡できること。
- OCR/LLM由来の値をGoogle Sheetsの数式として評価させないこと。
- LINEユーザー向け通知には内部例外、ARN、Secret名、Google Project情報などの詳細を含めないこと。
- AWS IAM権限は実行に必要なリソースへ限定すること。

## 受け入れ条件
- [ ] LINEにレシート画像を送ると受付メッセージが返る。
- [ ] レシート画像がS3へ保存され、`receipts/*` の公開URLで参照できる。
- [ ] 店舗名と合計金額を読み取れた場合、スプレッドシートへ登録日時、レシート日付、LINE表示名、LINEユーザーID、店舗名、カテゴリ、合計金額、画像URL、LINEメッセージIDが追記される。
- [ ] LINE表示名を取得できない場合でも、LINEユーザーIDを使って受付・登録処理を継続し、LINE表示名列は空欄で登録される。
- [ ] レシート日付に対応する `yyyy-MM` 形式の月別シートに追記される。
- [ ] 対象月のシートが存在しない場合、ヘッダー付きで新規作成される。
- [ ] 登録完了後、LINE Push Messageで登録結果が通知される。
- [ ] LINEに今月の全体合計を尋ねるテキストを送ると、現在月シートの合計金額合計がReply Messageで返る。
- [ ] LINEに今月の自分の合計を尋ねるテキストを送ると、送信者のLINEユーザーIDで絞り込んだ合計が返る。
- [ ] LINEにユーザー別合計を尋ねるテキストを送ると、LINEユーザーIDごとの合計一覧が返る。
- [ ] 対象外のテキスト質問では、対応できる質問範囲が案内され、スプレッドシートは更新されない。
- [ ] 許可されていないグループ、またはグループ以外からのテキスト集計質問では、Agent Core Runtimeを呼び出さず、集計できない旨が返る。
- [ ] 現在月シートが存在しない場合、今月の登録がまだない旨が返る。
- [ ] 店舗名または合計金額を読み取れない場合、スプレッドシートへ登録せず、再送依頼がLINEへ通知される。
- [ ] 同じLINEメッセージが再処理されても、既存のLINEメッセージIDを検出して二重登録されない。
- [ ] S3画像が90日後に削除される設定になっている。
- [ ] Google Sheets認証情報とLINE認証情報がSecrets Managerで管理されている。
- [ ] CDKでPublic API Gateway、Lambda、SQS、DLQ、S3、Agent Core Runtime、Secrets Manager、IAM Roleが構築できる。

## 補足
将来的には、精算対象外フラグ、分担率、月次集計、精算済み管理を追加し、同棲生活の月末精算をより少ない手作業で完了できる状態を目指す。

Agent Core IdentityはMVPでは使用しない。将来、利用者本人のGoogle権限でスプレッドシートやGoogle Driveへアクセスする必要が出た場合に導入を検討する。

`agentcore-sample` は参考用サンプルであり、本機能の実装では必要な考え方だけを参照する。不要な既存構成、VPC前提、Private API Gateway前提、複数Agent前提は採用しない。
