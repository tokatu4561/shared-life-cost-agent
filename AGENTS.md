## git 運用

- ブランチは作業単位で切る（例: `feature/article-crud`）。
- コミットメッセージは **「何を」より「なぜ」** を重視。

## ドキュメント管理（`docs/` 配下）

仕様書・設計書・調査メモなど、**コード以外のドキュメントは原則 `docs/` 配下に集約**する。
AGENTS.md / README.md はリポジトリ直下に置いたまま、それ以外の詳細ドキュメントは下記の構成で管理する。

## ドキュメント運用ルール

- docs/template.mdを雛形にする
- ER図などは Mermaid を第一選択（`.md` 内に直接記述）
- 仕様変更が発生したら **コードと同じ PR で `docs/` も更新**。ドキュメントの放置を許さない。
- 新規ドキュメントを追加したら **`docs/README.md` の目次に必ず追記**する。
- AGENTS.md は **リポジトリ全体の作業指針（変わりにくい原則）** を保ち、
  詳細・流動的な仕様は `docs/` 側に書いて参照する。
